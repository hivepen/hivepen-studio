Dashboard implementation guide

This document is the authoritative implementation reference for the Hivepen Studio dashboard. It covers every chart, minicard, visual pattern, and feasibility decision. It is meant to be read by an AI agent that will implement changes, not by end users.

Golden rule: prefer a smaller set of truthful charts over a larger set of impressive but weakly supported visuals. Every chart in this guide is either grounded in public Hive API data we already fetch, or explicitly labeled as infeasible until a trustworthy data source is added.

Feasibility legend

StatusMeaning
● DONEAlready implemented and rendering real data
● PHASE 1Build next — data is available or derivable with light aggregation
● PHASE 2Build later — feasible but needs additional API work or verification
● NEWNew addition requested in this revision
✕ AVOIDDo not implement — data is too weak or too inferential right now

Data sources we actually have

  • Account posts and comments for the selected range (getAccountPosts)
  • Reward history from get_account_history: author, curation, savings interest, witness, inbound transfers
  • Outgoing delegations from get_vesting_delegations
  • Global chain properties and median HIVE/HBD price
  • Wallet/profile-level account metrics (HP, voting mana, follower count, account age)
  • Post metadata: coverUrl, communityTitle, communityId, author, votes, comments
  • Hive avatar CDN: https://images.hive.blog/u/{username}/avatar
  • Community icon: https://images.hive.blog/u/{communityId}/avatar

Miniature charts for metric cards

Done

The MetricCard component accepts a media prop that replaces the icon box. Use it to embed a tiny ECharts canvas (80×40px) or a pure SVG sparkline showing trend context directly in the card. The children prop below the value remains for text context.

Rule: only use a minichart when you have at least 4 data points (e.g. weekly buckets) for the selected range. With fewer points, fall back to the icon. The minichart must render without axis labels or tooltips — it is decoration + at-a-glance trend, not an interactive chart.

MiniSparkline component — full implementation

MiniSparkline.tsx — new component TSX 
import { Box } from '@chakra-ui/react'
import { useEffect, useRef } from 'react'
import * as echarts from 'echarts/core'
import { LineChart, BarChart } from 'echarts/charts'
import { SVGRenderer } from 'echarts/renderers'
import { GridComponent } from 'echarts/components'

echarts.use([LineChart, BarChart, GridComponent, SVGRenderer])

const tokenVar = (token: string) =>
  `var(--chakra-colors-${token.replace(/\./g, '-')})`

function resolveCssVar(varExpr: string, el: HTMLElement): string {
  const match = varExpr.match(/var\((--[^)]+)\)/)
  if (!match) return varExpr
  return getComputedStyle(el).getPropertyValue(match[1]).trim() || '#888'
}

type MiniSparklineProps = {
  data: number[]
  colorToken: string       // e.g. 'green.solid'
  type?: 'area' | 'bar'
  height?: number           // default 36
}

export default function MiniSparkline({
  data, colorToken, type = 'area', height = 36,
}: MiniSparklineProps) {
  const ref = useRef<HTMLDivElement>(null)

  useEffect(() => {
    const container = ref.current
    if (!container || data.length < 2) return
    if (
      typeof navigator !== 'undefined' &&
      navigator.userAgent.toLowerCase().includes('jsdom')
    ) return

    const instance = echarts.init(container, undefined, { renderer: 'svg' })
    const resolve = (v: string) => resolveCssVar(v, container)
    const c = resolve(tokenVar(colorToken))
    const max = Math.max(...data, 0.001)

    instance.setOption({
      animation: false,
      grid: { top: 2, bottom: 2, left: 0, right: 0 },
      xAxis: { type: 'category', show: false,
               data: data.map((_, i) => i) },
      yAxis: { type: 'value', show: false, min: 0, max },
      series: type === 'bar'
        ? [{
            type: 'bar', data,
            itemStyle: {
              color: c, opacity: 0.85,
              borderRadius: [2, 2, 0, 0],
            },
            barWidth: '60%',
          }]
        : [{
            type: 'line', data, smooth: 0.4,
            symbol: 'none',
            lineStyle: { color: c, width: 1.5 },
            areaStyle: {
              color: {
                type: 'linear', x: 0, y: 0, x2: 0, y2: 1,
                colorStops: [
                  { offset: 0, color: c + '55' },
                  { offset: 1, color: c + '00' },
                ],
              },
            },
          }],
    })

    return () => { instance.dispose() }
  }, [data, colorToken, type])

  return <Box ref={ref} h={`${height}px`} w="full" />
}

Full component implementation

HpDelegationSankeyChart.tsx — complete implementation TSX 
import { Box, HStack, Stack, Text } from '@chakra-ui/react'
import { useEffect, useRef } from 'react'
import * as echarts from 'echarts/core'
import { SankeyChart } from 'echarts/charts'
import { TooltipComponent } from 'echarts/components'
import { SVGRenderer } from 'echarts/renderers'
import type { EChartsType } from 'echarts/core'
import type { DashboardDelegation } from './types'

echarts.use([SankeyChart, TooltipComponent, SVGRenderer])

type Props = {
  username: string
  delegations: DashboardDelegation[]
  ownHivePower: number          // wallet.metrics.hivePower
}

const tokenVar = (t: string) => `var(--chakra-colors-${t.replace(/\./g, '-')})`
const semVar   = (t: string) => `var(--chakra-colors-${t.replace(/\./g, '-')})`

function resolveCssVar(v: string, el: HTMLElement) {
  const m = v.match(/var\((--[^)]+)\)/)
  if (!m) return v
  return getComputedStyle(el).getPropertyValue(m[1]).trim() || '#888'
}

function fmt(n: number) {
  return new Intl.NumberFormat(undefined, {
    minimumFractionDigits: 0, maximumFractionDigits: 0,
  }).format(n)
}

export default function HpDelegationSankeyChart({ username, delegations, ownHivePower }: Props) {
  const chartRef = useRef<HTMLDivElement>(null)
  const instanceRef = useRef<EChartsType | null>(null)

  const totalDelegated = delegations.reduce((s, d) => s + d.hivePower, 0)
  const retained = Math.max(0, ownHivePower - totalDelegated)

  useEffect(() => {
    const container = chartRef.current
    if (!container || delegations.length === 0) return
    if (
      typeof navigator !== 'undefined' &&
      navigator.userAgent.toLowerCase().includes('jsdom')
    ) return

    const instance = echarts.init(container, undefined, { renderer: 'svg' })
    instanceRef.current = instance

    const ro = typeof ResizeObserver === 'function'
      ? new ResizeObserver(() => instance.resize()) : null
    ro?.observe(container)

    return () => { ro?.disconnect(); instance.dispose(); instanceRef.current = null }
  }, [delegations.length])

  useEffect(() => {
    const container = chartRef.current
    const instance = instanceRef.current
    if (!container || !instance || delegations.length === 0) return

    const c = (t: string) => resolveCssVar(t, container)

    // SOURCE NODE: the user
    // SINK NODES: each delegatee + "Retained" if any HP is not delegated out
    const nodes: { name: string; itemStyle: { color: string } }[] = [
      {
        name: `@${username}`,
        itemStyle: { color: c(tokenVar('teal.solid')) },
      },
      ...delegations.map((d) => ({
        name: d.delegatee,
        itemStyle: { color: c(tokenVar('teal.subtle')) },
      })),
    ]

    if (retained > 0) {
      nodes.push({ name: 'Retained HP', itemStyle: { color: c(semVar('bg.subtle')) } })
    }

    const links = [
      ...delegations.map((d) => ({
        source: `@${username}`, target: d.delegatee, value: d.hivePower,
      })),
      ...(retained > 0
        ? [{ source: `@${username}`, target: 'Retained HP', value: retained }]
        : []),
    ]

    instance.setOption({
      animation: false,
      tooltip: {
        backgroundColor: c(semVar('bg.panel')),
        borderColor: c(semVar('border.subtle')),
        borderWidth: 1,
        textStyle: {
          color: c(semVar('fg')),
          fontFamily: 'var(--chakra-fonts-body)',
        },
        formatter: (params: any) => {
          if (params.dataType === 'edge') {
            return [
              `<div style="font-weight:600;margin-bottom:4px">` +
                `${params.data.source} → ${params.data.target}</div>`,
              `<div>${fmt(params.data.value)} HP delegated</div>`,
            ].join('')
          }
          const pct = ownHivePower > 0
            ? ((params.value / ownHivePower) * 100).toFixed(1) + '%'
            : '—'
          return [
            `<div style="font-weight:600;margin-bottom:4px">${params.name}</div>`,
            `<div>${fmt(params.value)} HP · ${pct} of your stake</div>`,
          ].join('')
        },
      },
      series: [{
        type: 'sankey',
        layout: 'none',
        emphasis: { focus: 'adjacency' },
        left: 24, right: 120, top: 16, bottom: 16,
        nodeWidth: 12,
        nodeGap: 10,
        lineStyle: { color: 'source', opacity: 0.28, curveness: 0.5 },
        label: {
          color: c(semVar('fg.muted')),
          fontSize: 11,
          fontFamily: 'var(--chakra-fonts-mono)',
          formatter: (p: any) => {
            // For delegatee nodes: show avatar image is NOT possible in ECharts SVG
            // Instead: truncate long names and show HP value
            const name = String(p.name)
            const short = name.length > 14 ? name.slice(0, 13) + '…' : name
            return short
          },
          position: 'right',
        },
        data: nodes,
        links,
      }],
    }, { notMerge: true })
    instance.resize()
  }, [delegations, username, ownHivePower, retained])

  // React legend with avatars lives OUTSIDE the canvas
  if (delegations.length === 0) return null

  return (
    <Stack gap={3}>
      <Box ref={chartRef} h="14rem" w="full" />

      {/* Avatar legend — avatars cannot render inside ECharts SVG */}
      <Stack gap={1}>
        {delegations.slice(0, 8).map((d) => (
          <HStack key={d.delegatee} gap={2} justify="space-between">
            <HStack gap={2} minW={0}>
              <Box
                as="img"
                src={`https://images.hive.blog/u/${d.delegatee}/avatar/small`}
                alt={d.delegatee}
                boxSize="20px"
                borderRadius="full"
                border="1.5px solid"
                borderColor="teal.solid"
                objectFit="cover"
                filter="grayscale(20%)"
                flexShrink={0}
                onError={(e: React.SyntheticEvent<HTMLImageElement>) => {
                  e.currentTarget.style.display = 'none'
                }}
              />
              <Text fontSize="xs" color="fg.muted" fontFamily="mono" lineClamp={1}>
                @{d.delegatee}
              </Text>
            </HStack>
            <Text fontSize="xs" fontFamily="mono" color="teal.fg" flexShrink={0}>
              {fmt(d.hivePower)} HP
            </Text>
          </HStack>
        ))}
      </Stack>
    </Stack>
  )
}

Charts to explicitly avoid

Do not implement

These charts exist in the mock dashboard HTML or were requested elsewhere. Do not implement them until the data story is strong enough.

Curator profile radar
Requires a defensible "top-100 curator benchmark" dataset. We don't have it. The chart would be measuring the user against invented averages. Skip.
HP flow sankey (general)
Not the same as the delegation sankey. This would claim HP "flows" from voting → author rewards → savings — relationships we cannot honestly model from public data. Skip.
Witness rank timeline
Historical witness rank is not in our current data inputs. Requires dedicated historical snapshots. Skip until we have a real source.
RC usage breakdown pie
Publicly deriving RC consumption by operation category is non-trivial and easy to get wrong. Skip until a trustworthy upstream model exists.
HP accumulation projection
Math is easy; framing is the problem. Projections look like forecasts and erode trust when wrong. If ever built, must be a scenario tool with explicit user-set assumptions, never a dashboard fact view.
HIVE/HBD price correlation line
We only have the current median price, not a historical price series. Without that, the "correlation" chart would mix real reward data with a missing price axis. Skip until historical price source is added.

Shared implementation patterns

ECharts init pattern (all charts)

Every chart component follows the exact same init/cleanup/update split. Do not combine effects. Do not re-init on data change.

Standard chart component structure TSX 
// Effect 1: INIT — runs only when data presence changes (not on every data update)
useEffect(() => {
  const container = chartRef.current
  if (!container || !hasData) return
  if (navigator?.userAgent.toLowerCase().includes('jsdom')) return

  const instance = echarts.init(container, undefined, { renderer: 'svg' })
  instanceRef.current = instance

  const ro = typeof ResizeObserver === 'function'
    ? new ResizeObserver(() => instance.resize()) : null
  ro?.observe(container)

  return () => { ro?.disconnect(); instance.dispose(); instanceRef.current = null }
}, [hasData])  // ← only dependency is whether data exists

// Effect 2: UPDATE — runs whenever data or active state changes
useEffect(() => {
  const container = chartRef.current
  const instance = instanceRef.current
  if (!container || !instance || !hasData) return

  instance.setOption(option, { notMerge: true })
  instance.resize()
}, [chartData])  // ← data dependencies only
